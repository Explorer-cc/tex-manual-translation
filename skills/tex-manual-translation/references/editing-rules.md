# 编辑法则

## 黄金法则

编辑时只修改文本行——绝不要触碰 `\begin{}` / `\end{}` 等结构行。

进行行范围替换时，若替换范围内包含 `\begin{...}` 或 `\end{...}`，则**必须在替换正文中原样写回**——否则该标记将永久丢失，导致数十个连锁编译错误。

## 行号管理

每次编辑后，从工具响应中获取最新行号；切勿依赖记忆。

编辑会重新编号文件中的所有行。连续编辑时，始终以最近一次工具响应中的行号为准。

## 环境标记完整性检查

```bash
python scripts/check_env_balance.py <file.tex>
```

输出每个环境的 `\begin` / `\end` 计数。不平衡时报告具体行号，便于定位并恢复丢失的标记。

## 常见错误模式

1. **替换范围过宽**：SWAP 范围意外覆盖了 `\begin{document}` 或 `\end{verbatim}`，且替换正文中未写回 → 标记丢失
2. **跨行控制空格**：行尾 `\command\` + 下一行中文标点开头 → `\，` 类未定义控制序列
3. **记忆行号**：编辑后行号已变，仍用旧行号继续编辑 → 改错位置
